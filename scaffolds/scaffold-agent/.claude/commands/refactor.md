# Refactor Code

Refactor $ARGUMENTS following best practices and project conventions.

## Process

1. Analyze current code structure
2. Identify improvement opportunities
3. Plan refactoring steps
4. Execute changes incrementally
5. Verify functionality preserved

## Refactoring Types

- **Extract**: Extract functions, classes, or modules
- **Rename**: Improve naming for clarity
- **Simplify**: Reduce complexity and nesting
- **Consolidate**: Merge duplicate code
- **Modernize**: Update to modern patterns

## Guidelines

- Maintain existing functionality
- Make small, incremental changes
- Run tests after each change
- Follow project conventions
- Add documentation where missing

## Usage

```
/refactor src/utils/helpers.ts --extract
/refactor src/api --simplify
/refactor src/components --modernize
```
